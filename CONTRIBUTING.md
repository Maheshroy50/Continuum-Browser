## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in [Issues](https://github.com/Maheshroy50/Continuum-Browser/issues)
2. If not, create a new issue using the **Bug Report** template
3. Include as much detail as possible:
   - Your OS and version
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Features

1. Check existing [Issues](https://github.com/Maheshroy50/Continuum-Browser/issues) for similar suggestions
2. Create a new issue using the **Feature Request** template
3. Describe the problem you're trying to solve
4. Explain your proposed solution

### Code Contributions

1. **Fork** the repository
2. **Clone** your fork locally
3. Create a **feature branch**: `git checkout -b feature/amazing-feature`
4. Make your changes
5. **Test** your changes locally: `npm run dev`
6. **Commit** with clear messages: `git commit -m "Add amazing feature"`
7. **Push** to your fork: `git push origin feature/amazing-feature`
8. Open a **Pull Request**

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/Continuum-Browser.git
cd Continuum-Browser

# Install dependencies
npm install

# Run in development mode
npm run dev

# Build for production
npm run build
```

## Code Style

- Use TypeScript for all new code
- Follow existing code patterns
- Run `npm run lint` before committing
- Keep commits small and focused

## Pull Request Guidelines

- Reference any related issues
- Include screenshots for UI changes
- Update documentation if needed
- Ensure CI passes

## Questions?

Feel free to open a [Discussion](https://github.com/Maheshroy50/Continuum-Browser/discussions) if you have questions!
